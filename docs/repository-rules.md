# Repository Rules and Guidelines

## リポジトリの命名規則

- **一貫性**: 明確で一貫した名前付けを行う。
- **プレフィックス**: `lib-`, `app-`などのプレフィックスを使用。

## ディレクトリ構造

- `src/`: ソースコード
- `tests/`: テストコード
- `docs/`: ドキュメント
- `scripts/`: ツールやスクリプト

## コミットメッセージ

- **フォーマット**: `Add`, `Fix`, `Update`などのプレフィックスを使用。
- **例**: `Add authentication feature`、`Fix login bug`

## プルリクエスト

- **レビュー**: PRは必ずレビューを依頼する。
- **説明**: 変更内容と理由を詳細に記載する。
